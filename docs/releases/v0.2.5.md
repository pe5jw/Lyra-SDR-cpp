# Lyra v0.2.5 — VAC mic fix

A focused fix release for transmitting through a **Virtual Audio Cable**
mic (SSB voice or any mono source on a VAC), on top of v0.2.4's VAC + Profiles.

## Fixed

- **VAC mic input is now mono-combined (I=Q=L+R) before the transmitter.**
  Previously the captured cable was fed to the modulator as raw stereo
  (left→I, right→Q), so a mono mic that landed on the right channel reached
  the SSB modulator as silence → no TX audio, even though the same cable
  worked in other software. Lyra now sums the channels to mono so the mic
  reaches the modulator regardless of which channel it's on.

  This matches a VAC running in mono (the common voice-mic setup). There's
  a new **"Combine input (mono)"** checkbox in **Settings → Audio → VAC1**
  (default **ON**) if you ever need to feed true stereo I/Q over the cable
  instead.

## Notes

- If your other software used a higher VAC **TX gain**, set Lyra's
  **Settings → Audio → VAC1 → TX gain** to match (Lyra defaults to +3 dB).
- For transmitting through a VAC, the mic source must be **"PC Soundcard
  (VAC1)"** (Settings → TX), and VAC1 must be enabled with its **input
  device** set to the cable your audio arrives on.

## Install

Download **Lyra-Setup-0.2.5.exe** and run it. Windows 10 (1809+) / 11,
64-bit. Existing settings carry over.

## Hardware / scope

Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus SSB and digital
(TCI and VAC) transmit.
