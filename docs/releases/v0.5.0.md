# Lyra v0.5.0

**Hardware / scope:** Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus
CW / SSB / AM / DSB / SAM / FM and digital (TCI / VAC) transmit. **Windows
only** for now (Linux / macOS planned, not yet buildable).

The headline of this release is a full **DX-cluster spots subsystem** — three
spot sources, mode/region filters, clutter controls, and colour-coding right on
the panadapter — plus a big **CW RX-decoder** overhaul that copies far cleaner
in a crowded band.

## DX-cluster spots — three sources, on the panadapter

Spots from any source land on the panadapter as colour-coded callsign markers.
Click one to **tune to it and grab the call into the CW-console contact row**.
Everything lives in **Settings → Network → DX-cluster spots**.

- **Three sources, one list.** (1) **TCI** — spots pushed in by a logger /
  cluster app (e.g. SDRLogger+). (2) **SpotHole** — a standalone internet
  aggregator (cluster + POTA + skimmer, all modes); no logger needed, internet
  only. (3) **DX-cluster telnet** — connect to a specific node (VE7CC /
  DXSpider / AR-Cluster) with host, port, a login call (defaults to your
  MYCALL, or a club / friend / node call), and optional login commands. Each
  source has its own on/off.
- **Filters (apply to every source).** *Show modes* — e.g. `CW`, `SSB`, `DIGI`
  (family-aware; blank = all). *Show regions* — continent codes (NA/EU/AS/SA/
  AF/OC) and/or country codes like `US`, `DE` (the same codes shown on the
  markers); e.g. `US,EU` = US **plus** all of Europe.
- **Clutter controls** keep a busy band readable: a **global "show at most"**
  cap, and a **per-frequency cap** that collapses an FT8 pile-up into a few
  markers plus a **"+K"** badge.
- **Colour-coding.** Colour markers by **mode**, **region**, or **country**, or
  pick a **single custom colour**, with an optional legend. A freshly-arrived
  spot wears a **new-spot colour** for a few seconds so you notice it, then
  settles. Your own callsign is highlighted (with an optional toast) when it's
  spotted.

> Tip for telnet nodes: many default to **RBN CW skimmer spots only**. On
> VE7CC, add login commands like `set/ft8;set/ft4` to also receive FT8/FT4
> (and `set/nocw` to quiet the CW firehose). SSB spots are human-posted, so
> they're naturally sparse.

## CW RX decoder — cleaner copy in a crowded band

The CW decoder was reworked to read much closer to a dedicated decoder, even
during a contest or Field Day pile-up.

- **Narrow detection filter** (80 / 100 / 120 / 150 Hz) isolates **one** signal
  in the passband — the key to clean letter/word spacing when several CW
  signals sit close together. ~80 Hz for slow/clean signals, ~100–120 for
  typical speeds.
- **Auto level** sets the decoder's own squelch/threshold and adapts as
  conditions change (a small live readout shows the values).
- The panel was tidied — everyday controls up front, **Seek / DSP filter / AFC
  range** tucked into an **Advanced** section — and the decode window now
  **auto-scrolls** to follow the newest text.

## Also in this release

- **DSP filter type** (Settings → DSP) — choose **Linear Phase** or **Low
  Latency** per mode family (opt-in).
- The **CW console** now scrolls when shrunk, so its macros and option rows stay
  reachable at any panel size.

## Notes

- **Windows-only** reality is stated in the README and the About chip — Linux /
  macOS are on the roadmap but not yet buildable.
- The spots subsystem is implemented natively in C++23 / Qt6.

## Install
