# Lyra v0.4.11

**Hardware / scope:** Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus
CW / SSB / AM / DSB / SAM / FM and digital (TCI / VAC) transmit. **Windows
only** for now (Linux / macOS planned, not yet buildable).

This release adds **Kenwood CAT rig control + serial PTT**, so loggers and
digital-mode apps can drive Lyra over a COM port or TCP — and rolls in the
direct-connect (APIPA) radio-discovery fix.

## Kenwood CAT — control Lyra from your logger / digital app

A new **Settings → CAT / Serial** tab lets software talk to Lyra as a Kenwood
rig (TS-480 / TS-2000), reading and setting **VFO frequency + mode** and
keying TX — the way you'd connect any app to a real Kenwood radio. This is in
addition to the existing TCI server, so apps now have **three ways** to
connect: TCI, CAT-over-COM, and CAT-over-TCP.

- **Two transports.** Each CAT instance can run over a **COM port** (with
  Baud / Data / Parity / Stop) *or* over **TCP** — and TCP needs **no
  com0com**: the app just connects to Lyra's host/port (e.g.
  `127.0.0.1:60000`). Multiple apps can share one TCP instance.
- **Two independent, labelled instances.** Run (say) a logger on one and a
  digital app on another, each on its own port, each with a free-text label
  for what it's for.
- **Rig-model toggle.** Report as **TS-480** (ID 020) or **TS-2000** (ID 019)
  to match the rig your app expects (Thetis-style profiles want TS-2000).
- Commands: `FA`/`FB` (frequency), `MD` (mode), `IF` (status), `FR`/`FT`,
  `RX`/`TX` (PTT), `ID`, `PS`, `AI` (auto-info). Data modes map the standard
  way (**USB-D → MD9**, **LSB-D → MD6**).

## Serial PTT input

A digital-mode app (WSJT-X, VarAC, fldigi, …) can key Lyra over a **serial
PTT line** — set the app's PTT method to **RTS** or **DTR** on a (virtual)
COM port, and Lyra transmits, which keys your rig. Configurable CTS/DSR line,
invert, fast low-jitter detection. Lives in the same **CAT / Serial** tab.

## Direct-connect / APIPA radio discovery

If your HL2 is wired straight to the PC with no DHCP, both ends fall back to
**link-local 169.254.x.x** addresses. Lyra now **auto-discovers** a radio on
such a link, so the scan finds it without needing **Add by IP**.

## Notes

- **Windows-only** reality is now stated up front in the README and the About
  chip — Linux / macOS are on the roadmap but not yet buildable.
- CAT is implemented natively in C++23 / Qt6, modeled on the openHPSDR Thetis
  CAT command coverage (the Kenwood CAT grammar is an open standard).

## Install

`dist/Lyra-Setup-0.4.11.exe` — Windows installer (bundles the Qt runtime and
DSP engine; no separate downloads).
