# Lyra v0.3.0 — VAC device-layer rebuilt + TX metering restored

A substantial release that **rebuilds the Virtual Audio Cable (VAC) audio
device layer** to match the reference architecture, fixes the long-standing
**VAC/TCI crash**, and restores the **TX MIC / ALC / leveler meters**.

## Headline

- **VAC now runs as one full-duplex PortAudio stream** (the reference design),
  replacing the previous two independent Qt audio streams. The old approach
  ran the cable's playback and capture on two separate PC clocks with no
  transmit gating — the root cause of the feedback, the broken Mute, and the
  crashes that VAC users hit. PortAudio (19.7.0) is bundled; nothing extra to
  install.

## Fixed

- **The VAC / profile-switch crash is fixed.** Switching profiles (or
  enabling/disabling VAC) could crash Lyra. There were two distinct
  use-after-free bugs on that path — one in the VAC engine teardown (the audio
  mixer + transmit pump could still touch the audio buffers as they were being
  freed) and one in the TCI server shutdown (a dropped client could be
  dereferenced after deletion). Both are fixed; profile flips, VAC
  enable/disable, and stop/start are now stable.

- **RX audio is muted out of the VAC during transmit.** On key-down the
  receiver audio no longer bleeds into the VAC output (the "open-mic monitor"
  feedback some users heard, especially with the output pointed at real
  speakers). This is the reference's MOX-gating behavior, driven off the
  transmit edge.

- **TX MIC / ALC / leveler meters read live again.** The MIC, ALC, and leveler
  meters had been showing "—" (a regression from the transmit-engine rebuild).
  They now read live from the transmit DSP, so you can gain-stage VAC and SSB
  transmit by watching your actual input level and ALC action.

## Changed

- **Settings → Audio → VAC1 has a "Driver" picker.** You now choose the audio
  backend (host API — WASAPI / DirectSound / MME / WDM-KS), and the Output /
  Input device lists show that backend's devices — the same model the
  reference uses. WASAPI is the right choice for virtual cables (VB-Audio /
  VAC). Existing VAC device selections carry over.

- A device change while **Auto-enable for digital modes** is on now re-opens
  the cable correctly.

## Notes

- For transmitting through a VAC, the mic source must be **"PC Soundcard
  (VAC1)"** (Settings → TX), and VAC1 must be **enabled** with an **input
  and output device** selected (both are required in this release).
- Digital-mode users (WSJT-X / MSHV / FLDigi) route receive audio to the VAC
  **Output** device and transmit audio from the **Input** device, with PTT via
  TCI or VOX in the digital app.

## Install

Download **Lyra-Setup-0.3.0.exe** and run it. Windows 10 (1809+) / 11,
64-bit. Existing settings carry over; an existing install is upgraded in place.

## Hardware / scope

Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus SSB and digital (TCI and
VAC) transmit.
