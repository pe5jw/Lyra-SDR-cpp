# Lyra v0.8.0

**Hardware / scope:** Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus
CW / SSB / AM / DSB / SAM / FM and digital (TCI / VAC) transmit. **Windows
only** for now (Linux / macOS planned, not yet buildable).

This release is a batch of workflow and connectivity wins: a profile can now
**launch its digital-mode app** for you, CW gains **external-keyer / Winkeyer
and serial-key** support, the digital-audio bridge gets **tunable latency**,
and virtual COM ports finally **auto-list** for CAT. Plus a handful of fixes
that testers hit in the field.

## Companion-app launch on profile select (Settings → Profiles)

Pick your **VarAC** profile and VarAC starts; pick **MSHV** and MSHV starts.
A profile can optionally launch a companion digital-mode program when you
**explicitly select it** (from the front Profiles dock or the **Load**
button).

- **Enable + Browse** to the program (`.exe` / `.bat` / `.cmd`), add any
  **arguments**, and set a **start delay** so Lyra's CAT server + VAC come up
  first. **Test launch** tries it right now.
- It fires **only on an explicit pick** — never on the mode-family
  auto-recall, never at startup — and it **never closes** the app when you
  switch away (you close it).
- **Per-PC only:** the launch binding lives on your machine, is **never**
  written into the profile, and is **never** exported in a shared `.lyra`. A
  shared profile carries the DSP chain only — never "run this program."

## CW — external keyer / Winkeyer + serial key (Settings → CW)

- **External keyer / Winkeyer:** plug its **KEY output** into the HL2 KEY
  jack and turn **Iambic OFF** — the keyer does its own timing, so the radio
  passes the key straight through. (A Winkeyer's USB/COM port is for its own
  config app, not Lyra.) A short note in Settings → CW spells this out.
- **CW key over a COM port** *(new, off by default)*: a straight key, bug, or
  external keyer's key line wired to a serial-port **CTS/DSR** pin can key
  Lyra's CW. Enable it, pick the port + key line, in **Settings → CW**. For a
  paddle or high-speed sending the KEY jack is still the crisper route.

## Tunable digital-audio (VAC) latency (Settings → Audio)

The Virtual Audio Cable bridge now exposes a **Buffer size** and **ring
Latency** (with a live overflow/underflow monitor), so you can trim the
end-to-end delay for faster VarAC / ARQ turnaround — or keep the safe
defaults for SSB. The setting rides **per profile**, so a tight digital
profile can run low-latency while your voice profile stays relaxed.

## Virtual COM ports auto-list for CAT (Settings → CAT / Serial)

com0com / VSPManager virtual ports now appear in the CAT and serial-PTT port
pickers automatically (they were previously invisible to Lyra even though
they worked in the reference on the same PC). The COM-vs-TCP setup is also
clearer, with per-transport guidance and a tidier two-column layout.

## Fixes

- **Saved layouts restore exactly what you saved** — panels you'd closed no
  longer reappear on a named-layout recall / import.
- **USB-BCD (FTDI) amp band-data cable** is released cleanly on exit — no more
  "device not found" / cold-reboot needed to use it in another program, and
  the cable always shows in the picker.
- **Tuner** per-row delete is now a clear trashcan icon.
- **Docking is easier** — drop on a panel's centre to stack as a tab, and the
  drop preview names what each drop will do.

---

*Every feature landed gated by an operator HL2 bench test. The new serial
CW-key input ships **off by default**; enable it in Settings → CW.*
