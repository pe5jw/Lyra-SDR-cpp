# Lyra v0.12.2

**Hardware / scope:** Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus
CW / SSB / AM / DSB / SAM / FM and digital (TCI / VAC) transmit. **Windows
only** for now (Linux / macOS planned, not yet buildable).

A quick fix release for the space-bar PTT — worth grabbing if you key up
with the keyboard.

## Space-bar PTT is now a toggle

Some of you reported the space-bar PTT feeling unstable or intermittent —
occasionally getting stuck transmitting. That's fixed, by changing how it
works:

- **Press space once → transmit on. Press space again → transmit off.**

This matches Thetis and the PTT-lock on a normal transceiver, so there's
nothing new to learn. It also removes the underlying cause of the
stuck-transmit: the old "hold to talk" behaviour depended on the app
reliably seeing the key *release*, and if Lyra lost focus mid-transmission
(you Alt-Tabbed, a pop-up opened, you clicked into another window) that
release could go missing and leave the radio keyed. A toggle doesn't depend
on the release at all, so that can't happen anymore.

Details that keep it well-behaved:

- **Holding space doesn't do anything odd** — a held key won't rapid-cycle
  transmit on and off.
- **Turning transmit *off* always works** — space can always un-key you, so
  you're never stranded on the air.
- Space still types a normal space into a text field (frequency entry, the
  CW console, a callsign box) — it only keys PTT when you're not typing.
- The space-bar and the on-screen **MOX** button now control the exact same
  thing, so either one can un-key the other.

(Space-bar PTT is on by default; you can turn it off in **Settings →
Hardware**, next to the hardware-PTT option.)
