# Lyra v0.6.0

**Hardware / scope:** Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus
CW / SSB / AM / DSB / SAM / FM and digital (TCI / VAC) transmit. **Windows
only** for now (Linux / macOS planned, not yet buildable).

The headline of this release is a new **TX power-control system** — a
dedicated **PA Gain** Settings tab that calibrates your drive in real watts
per band and **auto-tunes a watts-output cap** to protect a low-drive
amplifier. Plus an FM transmit fix.

## Per-band TX power calibration + auto-tuning watts cap (Settings → PA Gain)

A new **Settings → PA Gain** tab gives you proper per-band TX power control,
ported faithfully from Thetis and **PureSignal-safe** (the gain runs inside
the TX DSP where PureSignal taps its reference).

- **PA Gain By Band table.** One number per band (default 100 = neutral).
  Each HL2 — and each band — makes different power, so nothing is computed:
  key a band into a dummy load, watch the PWR meter, and nudge its number
  until the dial reads true watts. Lower tames a hot band, higher pushes a
  weak one. Your drive % stays the dial; this calibrates what it *means*.
- **Watts output cap that auto-tunes per band.** Set one **Max Output (W)**
  value to protect a low-drive amp. Key **TUN** on each band and Lyra walks
  the power **up from below** until the meter reaches your cap, then **locks
  that band exactly at it** — a green ✓ in the new **Cap tuned** column.
  SSB and the other modes then hold each tuned band at the cap automatically,
  capping voice peaks without chasing them. Because it only ever approaches
  from below, the output never overshoots — safe for a solid-state amp where
  a fraction of a watt over matters.
- **Live "Cap tuned" indicators** — green ✓ for a band locked at the cap, red
  — for one not yet tuned, so you can see at a glance which bands still need a
  TUN pass (they turn green as you key each one).

> ⚠ **Calibrate into a dummy load with your amplifier out of line.** Tuning a
> band walks the power up to find your cap — verify every band settles
> correctly first, then put your amp back in.

The old **"Max TX drive %"** control is retired — the watts cap supersedes it.

## FM transmit no longer splatters at wide TX bandwidths

The FM transmit bandpass is now sized from the FM signal's own occupied
channel (Carson, ±(deviation + 3 kHz)) instead of inheriting a wide SSB / ESSB
TX filter width — so setting an ESSB-width filter no longer widens the FM
signal into splatter.

## Notes

- **Windows-only** reality is stated in the README and the About chip — Linux /
  macOS are on the roadmap but not yet buildable.
- The TX power model is implemented natively in C++23 / Qt6.

DSP engine; no separate downloads).
