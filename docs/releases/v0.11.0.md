# Lyra v0.11.0

**Hardware / scope:** Hermes Lite 2 / 2+ over HPSDR Protocol 1. RX, plus
CW / SSB / AM / DSB / SAM / FM and digital (TCI / VAC) transmit. **Windows
only** for now (Linux / macOS planned, not yet buildable).

The headline is **frequency calibration** — measure your radio's oscillator
error against a time station and make the dial read true. This release also
adds a one-step per-band power calibration to the amp watts-cap, and makes
your **CTUN** and **NR-C** settings stick across restarts.

## Frequency calibration (header → Options → Freq Cal)

Every radio's reference oscillator is a hair off, so a signal on a *known*
frequency lands a few hertz from where the dial says. Lyra now measures that
error against a rock-solid reference — a time station like **WWV**, **WWVH**
or **CHU** — and nudges every RX **and** TX frequency by a tiny factor so the
dial reads true.

- **A guided instrument.** Open **Options → Freq Cal**. Tap a time-station
  chip — Lyra tunes it in USB, and the carrier marker walks across a
  centre-null scale as the big **"Hz off"** readout colours green toward
  zero. Press **Apply**.
- **Only stations you can hear.** The picker is filtered by your Hardware
  **Region**, so a US setup lists WWV / WWVH / CHU and hides the ones you'd
  never receive.
- **A no-frills version too.** **Settings → Calibration** runs the same
  measurement without the moving dial, plus manual entry, **Reset to 1.0**
  (always back to how the radio tunes today), and one-level **Restore
  previous**.
- **Re-calibrate any time.** Apply composes onto whatever correction is
  already in effect, so measuring again is exact — no need to reset first.

Most HL2 boards are within a hertz or two and never need this; it's there for
the operator who wants the dial spot-on. **`1.00000000` = no correction**, so
you can always get back to stock.

## Amp watts-cap: one-step per-band calibration

Setting up the auto-tuning amp watts-cap is now a single step. **Tune your
band up to full drive once** and the measured watts auto-fill the band's
**Full Output** reference — no more entering it by hand. The watts cap itself
now sits behind an explicit **"Enable Max cap (arm)"** so it can never
silently hold your power low, and each band gets a **Clear** button. The
PA Gain tab was also cleaned up for readability.

## CTUN + NR-C remembered across restarts

Two receive settings now come back the way you left them:

- **CTUN** re-engages at launch if it was on (locking at your last dial).
- **NR-C** re-enables if it was on **and** its profile still matches your
  sample rate.
- New **Settings → Noise → "Auto-load on startup"** picks which saved noise
  profile loads each launch, so NR-C comes up ready instead of an empty
  field. Set it to **None**, or turn the toggles off before closing, for a
  clean start.

Both are receive-only, so there's no on-air surprise — and a profile that no
longer matches your rate simply loads nothing and leaves NR-C off (with the
usual recapture hint).

---

**Upgrade note:** nothing about your existing operating changes on upgrade.
Frequency calibration defaults to **no correction** until you run it, the
amp watts-cap stays **off until you arm it**, and CTUN / NR-C only persist
the state you leave them in.
