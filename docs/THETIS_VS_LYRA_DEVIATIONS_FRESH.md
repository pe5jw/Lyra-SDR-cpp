## Executive Summary

- **Operator rate-table violation â€” TX block size doubled.** `kInSize=128` at `src/tx_channel.cpp:65`, `kBlockSize=128` at `src/tx_dsp_worker.h:97`, `kTciTxBlockSamples=128` at `src/tci_server.h:242`. Reference `ChannelMaster/cmsetup.c:106-111` `getbuffsize(48000)=64`. **Fix:** revert all three constants to 64; centralize as one public constant.
- **PureSignal v0.3 and mic_bias/mic_ptt blocked today â€” EP2 case-11 C1/C2/C3 hard-zero.** `src/hl2_stream.cpp:2350-2360` emits only C4; reference `networkproto1.c:1093-1098` packs preamps + mic_trs/bias/ptt on C1, line_in_gain + puresignal_run<<6 on C2, user_dig_out on C3. **Fix:** compose C1/C2/C3 from atomics; puresignal_run on C2 bit-6 is forward-compat critical.
- **Verify mic-source routing is actually flipping on TCI keydown.** Wire lens reports current code at `src/tci_server.cpp:1287-1289` uses `wantTciAudio = useTciAudio || (prefs_->micSource()=="tci")` â€” i.e. the `,tci`-token gate is already removed and the predicate falls through when operator has Mic Source=tci in Prefs.
